# Security Policy

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue,
please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

Email: security@reactor.cloud

Please include:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Any suggested fixes (optional)

### What to Expect

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 7 days
- **Resolution Timeline**: Depends on severity, typically 30-90 days

### Scope

This policy applies to:

- All code in this repository
- The Reactor.cloud hosted service
- Official SDKs and CLI tools

### Recognition

We appreciate responsible disclosure. With your permission, we will:

- Credit you in the security advisory
- Add you to our security acknowledgments

### Out of Scope

- Social engineering attacks
- Denial of service attacks
- Issues in third-party dependencies (report to the upstream project)

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < 1.0   | :x:                |

## Security Updates

Security updates are released as patch versions and announced via:

- GitHub Security Advisories
- Release notes
- The `security@reactor.cloud` mailing list (subscribe by emailing with subject "subscribe")
